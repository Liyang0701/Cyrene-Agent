# 变更：微信支持多个 iLink Bot 连接账号

## Why

Cyrene 当前只保存一份微信凭据并运行一个 `ILinkBotAdapter`。再次扫码会覆盖原账号，应用启动、状态展示、退出登录和消息路由也都假设微信只有一个连接身份，无法让多个扫码绑定者通过各自独立的 iLink Bot 账号同时使用同一个 Cyrene 实例。

## What Changes

- 以稳定 `ilinkBotId` 管理多个微信连接账号，每个账号独立保存备注、启用状态、加密凭据、绑定者、连接状态和错误状态。
- 在微信 adapter 内建立账号连接池，同时运行多个独立长轮询连接；单账号失败、重登、停用或删除不影响其他账号。
- 以“连接账号 + 绑定者”构成微信对话身份，隔离历史、长期记忆、关系、任务、上下文令牌、权限和日志。
- 每个账号只接受扫码返回的 `ilinkUserId` 本人消息；其他联系人消息在进入 LLM、工具、历史和记忆前静默丢弃。
- 入站回复强制沿用原连接账号；主动消息和任务投递必须显式指定账号与目标。
- 同一对话串行处理，不同账号有限并发并公平排队；初版全局最多同时执行两个 LLM 请求。
- 所有微信账号共享全局唯一的活动角色与推理服务；角色切换期间完成已开始回复并暂存尚未执行的消息。
- 提供多账号设置页和逐账号添加、备注、启停、重连、重新扫码、退出登录及删除操作。
- 自动迁移旧版单账号凭据和历史，迁移失败时保留旧版兼容路径，不让微信整体不可用。

## Out of Scope

- 每个微信账号选择不同角色。
- 多人联系人审批、亲友共享或“所有联系人可用”模式。
- 微信原生语音条；继续使用已验证的 M4A/WAV 文件回复。
- 跨设备导出或迁移微信登录凭据。
- 改造飞书、搜索、地图或游戏功能。
- 为超过 3 个账号提供正式压力承诺。

## 可复用方案

腾讯官方 `openclaw-weixin` 已提供多账号索引、逐账号凭据文件、逐账号同步游标、复合 context token、账号级日志和显式 outbound account 路由。这些结构与协议经验可以复用。

不直接整套接入，因为 Cyrene 使用自己的 ChannelManager、Electron IPC、macOS `safeStorage`、角色私有状态、工具权限、任务系统和角色切换事务；官方实现也不满足“只允许扫码绑定者本人”“不继承桌面权限”和本地状态隔离要求。

## Success Criteria

- 可配置 3 个微信连接账号，至少 2 个同时在线并正常收发文字、图片、语音文件及表情。
- 两个账号同时发消息时，回复、上下文、历史、记忆、关系、任务、TTS 和权限不串线。
- 非绑定者消息不进入 LLM、工具、历史或记忆，也不收到自动回复。
- 单账号断线、凭据失效、停用、重登和删除不影响其他账号。
- 应用重启后自动恢复所有已启用且凭据有效的账号。
- 角色切换期间连接保持在线，旧回复完整结束，排队消息在切换后由正确角色处理。
- 旧版单账号数据可自动迁移且不要求重新扫码；失败时有安全兼容和可恢复状态。
- 凭据不出现在日志、诊断包、普通设置导出或角色包中。
