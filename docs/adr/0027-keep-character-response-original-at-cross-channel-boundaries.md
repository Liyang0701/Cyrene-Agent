# 在跨渠道边界保留角色回复原文

## 决策

定稿后的 **Character Response（角色回复）** 是桌面聊天、语音通话和外部渠道中唯一的业务发言。只有它可以进入 TTS、聊天历史、长期记忆、Relationship Event、语气分析、表情包、Semantic Action 和后续模型上下文。

**Translation Overlay（翻译附注）** 是由唯一的 Active Character Response 服务生成的展示产物，不是第二句角色发言。桌面与通话界面把它渲染为原文下方的从属文字；纯文本渠道会在原文成功投递后，另发一条带有“仅供理解，非角色发言”明确标记的附注。由于微信等渠道没有可编辑的富文本附注能力，这在传输层会是另一条消息，但在领域语义、历史、记忆和 TTS 中始终不是另一轮角色回复。

这项决策拒绝“直接生成双语回复”的便利做法：那会把展示译文悄悄教给记忆和 TTS，使系统错误地把附注当作角色的第二次发言。缓存、翻译 Provider 和超时等机制只保留在统一服务内部，渠道适配器不得各自实现翻译业务规则。
