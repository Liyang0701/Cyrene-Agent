# Discover built-in and locally imported Character Packages

The first version’s Character Registry contains read-only packages shipped with the application, with Cyrene as the default, plus packages imported from local folders into application-owned storage. Every package has a stable `id`, `schemaVersion`, and `version`; remote installation, an online marketplace, and direct Character Card V3 runtime support are deferred so that download trust, licensing, update sources, and external format constraints do not become prerequisites for proving safe local character switching.
