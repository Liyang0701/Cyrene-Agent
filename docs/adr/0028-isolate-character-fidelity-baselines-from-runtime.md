# ADR 0028：将角色还原度基线与运行时隔离

## 决策

把当前工程版星野冻结为只读、仅本机保存的 `Fidelity Baseline`：只包含声明过的文本角色内容及每个文件的 SHA-256。它不是 Character Package，不会加入 Character Registry，也没有 Character State Root、关系、记忆、Live2D、音色或当前角色的运行时所有权。候选包只从独立目录读取，不导入、不切换、不覆盖当前可运行包。

盲测公开文件只保存随机 A/B 回答；版本映射、模型配置、随机 seed 与内容摘要单独保存在权限更严的私有元数据中。用户评分只引用 A/B；汇总阶段才可以结合私有映射。生成器只调用 `127.0.0.1` / `::1` 上的 HTTP OpenAI 兼容服务，不携带云端凭据、工具、远端回退或遥测。

## 后果

这使工程版星野成为可追溯的比较基准，同时不会产生第二个角色身份或污染任何用户状态。自动规则只能报告明确的串角、语言、翻译和工具结果错误；是否更像星野仍由匿名人工盲测决定，且报告永不自动接受或替换角色包。
