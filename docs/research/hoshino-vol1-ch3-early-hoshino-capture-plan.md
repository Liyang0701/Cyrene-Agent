### E10-A / E10-B 的信息增益

两段必须连同梦前辈的回应保留，不能只摘星野的强硬句：

- E10-A 证明早期星野的愤怒不是天生冷漠，而是对无力、失败、恶意大人和无法改变现实的集中反应。
- 回忆中的星野用更直接、较少拖长音与玩笑的严肃语体否定奇迹，并要求梦前辈正视现实。
- E10-B 让“拒绝希望”与后果相连：她承认自己在烦躁中迁怒、先离开，本想很快道歉，却再也找不到梦前辈。
- 这些只能支持“星野如何理解和回顾该事件”；在正式客户端证据取得前，不得把 OCR 文本当逐字原句，也不能把她的自责直接写成客观死因。

### E12-A 的信息增益

- 早期星野先询问白子的身份与处境，发现失忆后决定把她带进室内。
- 她主动给白子围上自己的围巾，以“没有总比有好”式的实用理由隐藏照顾行为。
- 白子觉得温暖后，星野出现 `うへ～`、`おじさん` 和亲昵称呼；野宫随即注意到她的笑法和自称变化。这是“严肃早期语体 → 熟悉的大叔语体”的近距离转折证据。
- 该段同时包含行动、他人即时反应和星野回避解释，价值高于脱离上下文单摘一句口癖。

## 预计产出与验收

首批三段预计可产生约 **19～26** 个高价值星野日文候选；合并纯停顿、重复口癖和依赖上下文的短句后，预计适合入库 **12～18** 个语义单元。第二批预计再提供约 **14～20** 个候选，最终是否拆分仍由逐帧复核决定，不能以达到数量 Gate 为目标强行切句。

取得录屏后的自动化流程：

1. 保留原始文件，不移动或覆盖用户下载文件；复制到 Git 外 evidence 目录。
2. 记录原始文件名、日期、时长、分辨率、编码和 SHA-256。
3. 逐帧确认日服说话者栏、完整日文对话框和必要前后文；OCR 只作辅助。
4. 再逐帧检查 P57 同一来源单元，确认完整国服 UI、说话者与本地化文本。
5. 只在日中两侧都完成复核后建立稳定 `unitId` / `pairId`；没有中文对应的日文仍可作为未配对 A1 证据保存。
6. 把结论继续分成 `official fact → official dialogue → personality inference → language feature → assistant adaptation`，本阶段不修改 `identity`、`soul` 或运行时代码。

通过标准：第 10 话至少能用完整上下文回答“她为何否定希望、事后如何理解自己的行为”；第 12 话至少能回答“她如何迎接白子、何时开始采用大叔语体、如何把照顾转化为前辈与委员会关系”。
